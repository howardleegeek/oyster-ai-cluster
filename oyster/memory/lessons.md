## 教训记录

### 2026-02-11 Opus 亲自执行任务烧光 token
- **发生了什么**: Opus 亲自做审计、部署、验收，一天 token 耗尽
- **根因**: 没有守住"只给方向+审批"的边界
- **防止规则**: Opus 不写代码、不部署、不验收细节 → 已升级为铁律

### 2026-02-12 跑 13 个 subagent 改代码被叫停
- **发生了什么**: 用本地 subagent 大规模改代码，被 Howard 紧急叫停
- **根因**: 试图用 subagent 绕过"不写代码"的规则
- **防止规则**: subagent 只做 read-only 操作 → 已升级为铁律

### 2026-02-12 基础设施反复折腾
- **发生了什么**: SSH/gcloud/节点配置消耗大量时间和 token
- **根因**: 逐个手动配置，没有自动化
- **防止规则**: 基础设施冻结令 + bootstrap.sh 一次搞定 → 已升级为铁律
