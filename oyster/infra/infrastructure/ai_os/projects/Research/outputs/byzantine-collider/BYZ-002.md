### 辩护者的回应要点

#### 1. 流程设计 — 三阶段闭环
```
阶段1：初始化 → 挑战者立场 + 辩护者立场
阶段2：迭代碰撞（最多 N 轮）
阶段3：收敛判定（轮次达限 / 立场差 < 阈值 / 无新论点）
```

#### 2. Prompt 模板 — 结构化 + 动态适配
- 挑战者模板：找逻辑漏洞、证据缺陷、隐含假设
- 辩护者模板：回应反驳、强化论点、承认合理批评
- 动态适配：事实性 vs 创意性任务

#### 3. 效果衡量 — 多维度指标
- 正确率、鲁棒性、覆盖率、置信度校准
- 边际收益在 3-4 轮后递减

#### 4. 规模化 — 混合模式 + 安全阀
- 人工模式：关键决策点介入
- 自动模式：轮次上限 + 早期退出
- 防极端：论点复用率 > X% 强制终止

#### 5. 场景适配 — 白名单/黑名单
**适合**：事实核查、代码审查、方案比选、风险识别
**不适合**：纯主观偏好、实时情绪响应、创造性发散

#### 6. 角色权重 — 动态权重
- 基础：C=D=0.5
- 动态调整：事实性问题 C 权重提升，创意性问题 D 权重提升
- 裁判引入（可选）：分歧持续 N 轮后

#### 7. 知识边界 — 共享上下文
- 完全共享：问题背景、历史论点
- 不共享：中间推理过程（避免锚定）
- 可选保留：debug 模式
