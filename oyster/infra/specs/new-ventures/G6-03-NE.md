## 目标
引入异步任务处理机制，以提高 API 的响应速度和吞吐量。

## 技术方案
1. 使用 Celery 和 Redis 来实现异步任务队列。
2. 修改现有的 API 路由，使其将耗时任务提交到 Celery 队列。
3. 在 `api.py` 中添加新的端点以查询任务状态。
4. 编写单元测试以验证异步任务的提交和执行。
5. 在 `tests/test_api.py` 中添加相应的测试用例。

## 约束
- 在现有代码库内修改，不新建超过3个文件。
- 不留 TODO/FIXME/placeholder。
- 编写基础测试，确保异步任务机制能够正确处理任务。

## 验收标准
- [ ] API 端点能够在 100 毫秒内响应，即使任务在队列中。
- [ ] pytest 全绿，确保新功能不影响现有功能。
- [ ] 异步任务能够在 5 秒内完成（对于典型任务）。
