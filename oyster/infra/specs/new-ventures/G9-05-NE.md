## 目标
实现自动化的 CI/CD 流程。

## 技术方案
1. 使用 GitHub Actions 配置 CI/CD 流程，包括代码检查、测试和部署步骤。
2. 在代码检查步骤中运行 `flake8` 和 `black` 以确保代码质量和格式。
3. 在测试步骤中运行 `pytest` 并生成测试报告。
4. 在部署步骤中，将应用部署到指定的服务器或云平台。
5. 在 `deployment.py` 中编写部署脚本。
6. 编写基础测试以验证 CI/CD 流程的各个步骤。

## 约束
- 在现有代码库内修改，不新建超过3个文件。
- 不留 TODO/FIXME/placeholder。
- 写基础测试。

## 验收标准
- [ ] CI/CD 流程在每次提交时自动触发。
- [ ] 代码检查和测试步骤通过。
- [ ] 应用成功部署到目标环境。
- [ ] pytest 全绿。

## 不要做
- 不重构现有代码。
- 不改 UI/CSS。
