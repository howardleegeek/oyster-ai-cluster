## 任务: S9-2 内存管理优化

### 背景
长对话和多会话可能导致内存占用过高。

### 具体要求

#### Android
1. 会话列表: 使用 RecyclerView ViewHolder 回收 (确认已实现)
2. 图片缓存: 使用 LRU 缓存，限制大小
3. 消息列表: 只保留可见 ±50 条消息在内存，其余分页加载
4. 退出会话时释放 SSE 连接和大对象

#### iOS
1. 长列表使用 `LazyVStack` (确认已实现)
2. 图片缓存管理 (NSCache)
3. 消息分页: 初始加载 50 条，上滑加载更多
4. 会话切换时清理上一个会话的消息

### 文件
- Android: `ChatActivity.java`, 会话相关
- iOS: `ChatView.swift`, `ChatViewModel.swift`

### 验收标准
- [ ] 500+ 条消息的会话不卡顿
- [ ] 内存使用保持在合理范围 (< 200MB)
- [ ] 上滑加载历史消息流畅
- [ ] 编译通过 (双端)
